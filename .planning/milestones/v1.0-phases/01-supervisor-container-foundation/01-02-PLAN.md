  <behavior>
    - TmuxService.createSession starts a tmux session with the command as initial process (avoids send-keys race condition)
    - TmuxService.sendKeys sends text to a session and presses Enter
    - TmuxService.capturePane returns visible terminal content (optionally with scrollback)
    - TmuxService.killSession captures scrollback BEFORE destroying the tmux window (locked decision)
    - TmuxService.setSessionHooks sets pane-exited hook to POST to /internal/session-event
    - TmuxService.enableOutputPipe starts pipe-pane to /data/sessions/{id}/live-output
    - SessionManager.create creates a tmux session running `claude` with given flags, generates session ID, stores metadata
    - SessionManager.list returns all sessions with current status
    - SessionManager.stop sends Ctrl+C to a session
    - SessionManager.kill captures scrollback, saves to /data/sessions/{id}/, destroys tmux session
    - SessionManager.archive stops the session, saves scrollback text + metadata JSON to /data/sessions/{id}/archive/
    - SessionManager.revive reads archived scrollback, starts new session with --continue flag feeding scrollback as context file
    - Revive is restore-only -- no optional prompt parameter (locked decision)
    - Session status detection is event-driven via tmux hooks (locked decision -- NOT polling)
  </behavior>
  <action>
1. **supervisor/src/services/tmux.ts** -- TmuxService class:
   - Use `child_process.execFile` with promisify for all tmux commands
   - `createSession(id, command, workdir?)`: `tmux new-session -d -s claudeos_{id} -x 200 -y 50 [-c workdir] {command}`. Pass command as initial process to avoid send-keys race (Pitfall 1 from research).
   - `setSessionHooks(id)`: Set `pane-exited` hook to POST to `http://localhost:3100/internal/session-event` with sessionId and event. This is the event-driven approach -- NOT polling.
   - `enableOutputPipe(id)`: `tmux pipe-pane -t claudeos_{id} "cat >> /data/sessions/{id}/live-output"` for real-time output streaming.
   - `sendKeys(id, text)`: `tmux send-keys -t claudeos_{id} {text} Enter`
   - `capturePane(id, scrollback?)`: `tmux capture-pane -t claudeos_{id} -p [-S -1000 for scrollback]`
   - `killSession(id)`: Capture full scrollback FIRST, then `tmux kill-session -t claudeos_{id}`. Return the captured scrollback.
   - `stopSession(id)`: `tmux send-keys -t claudeos_{id} C-c ""`
   - `listSessions()`: `tmux list-sessions -F "#{session_name}"`, filter for claudeos_ prefix. Parse output.
   - `hasSession(id)`: Check if tmux session exists via `tmux has-session -t claudeos_{id}`

2. **supervisor/src/services/session-manager.ts** -- SessionManager class:
   - Constructor takes `dataDir` and `TmuxService` instance, optional event emitter for status broadcasts
   - Maintain in-memory Map<string, Session> for active sessions
   - On startup, scan /data/sessions/ to rebuild state from disk (session metadata files)
   - `create(options)`: Generate unique ID (e.g., `ses_` + nanoid or crypto.randomUUID().slice(0,8)), create tmux session with `claude [--model model] [flags]`, set hooks, enable pipe, save metadata to /data/sessions/{id}/meta.json, emit status "active"
   - `list()`: Return all sessions from in-memory map
   - `getSession(id)`: Return single session or null
   - `stop(id)`: Send Ctrl+C via tmux, update status to "stopped"
   - `kill(id)`: Capture scrollback (via tmux.killSession), save scrollback to /data/sessions/{id}/scrollback.txt, remove from active sessions, emit status update
   - `archive(id)`: Stop the session, capture full scrollback, save to /data/sessions/{id}/archive/ with: scrollback.txt (raw text) and meta.json (session name, creation time, working directory, model, flags, archivedAt). Update session status to "archived". Per locked decision: no retention limit, kept forever.
   - `revive(id)`: Read archived scrollback.txt and meta.json. Create new tmux session with `claude --continue` flag, pass scrollback as context file. Revive is restore-only per locked decision -- no optional prompt parameter. User sends new input separately after session is running.
   - `handleSessionEvent(sessionId, event)`: Called by internal route when tmux hook fires. Update session status accordingly, emit status change via provided callback.
   - File I/O: Use atomic writes (write to temp file, then rename) per research recommendation.
   - Session directories: /data/sessions/{id}/meta.json, /data/sessions/{id}/scrollback.txt, /data/sessions/{id}/archive/scrollback.txt, /data/sessions/{id}/archive/meta.json, /data/sessions/{id}/live-output

   - Register under /api/v1/sessions prefix
   - Access SessionManager via Fastify decoration (decorator added in server.ts)
   - Routes with Zod schema validation (from Plan 01's schemas):
     - POST / -- Create session. Body: CreateSessionSchema. Response: 201 + SessionResponseSchema.
     - GET / -- List sessions. Response: SessionListResponseSchema.
     - GET /:id -- Get session. Response: SessionResponseSchema. 404 if not found.
     - POST /:id/stop -- Stop session. Response: { success: true }.

     - POST /:id/input -- Send input. Body: SendInputSchema. Response: { success: true }.
