1. **supervisor/src/types.ts** -- Define ALL TypeScript interfaces for the entire supervisor:
   ```
   SessionStatus = "active" | "idle" | "waiting" | "stopped" | "archived" | "zombie"
   Session { id, name, status, createdAt, workdir?, model?, flags?, pid? }
   SessionArchive { sessionId, name, scrollback (text), metadata: { createdAt, archivedAt, workdir, model, flags } }
   Secret { name, category?, tags?, createdAt, updatedAt } (value never exposed in list/response -- only via getSecret)
   SecretEntry { ...Secret, encrypted, iv, authTag } (internal storage format)
   ExtensionInstallMethod = "github-release" | "build-from-source" | "local-vsix"
   ExtensionInstallRequest { method, repo?, tag?, localPath? }
   ExtensionInstallState = "pending" | "downloading" | "installing" | "installed" | "failed"
   ExtensionRecord { id, name, version, method, state, installedAt?, error? }
   BootState = "initializing" | "setup" | "installing" | "ready" | "ok"
   HealthResponse { status: BootState, version: string, uptime: number }
   SupervisorSettings { reloadBehavior: "force" | "notification", ... }
   WsMessage (discriminated union): subscribe, unsubscribe, status, output
   ServerOptions { dataDir: string, isDryRun: boolean, port?: number }
   ```

2. **supervisor/src/schemas/common.ts** -- Shared Zod schemas:
   - Import from 'zod' (Zod 4 uses `import { z } from 'zod'`)
   - HealthResponseSchema with status enum, version string, uptime number
   - ErrorResponseSchema { error: string, statusCode: number }
   - PaginationSchema { limit: number, offset: number }

3. **supervisor/src/schemas/session.ts** -- Session Zod schemas:
   - CreateSessionSchema { name?: string, workdir?: string, model?: string, flags?: string[] }
   - SessionResponseSchema matching Session type
   - SessionListResponseSchema array of sessions
   - SendInputSchema { text: string }
   - CaptureOutputSchema { scrollback?: boolean }
   - Note: All schemas use Zod 4 syntax

4. **supervisor/src/schemas/extension.ts** -- Extension Zod schemas:
   - InstallExtensionSchema with discriminated union on method field:
     - github-release requires repo + tag
     - build-from-source requires localPath
     - local-vsix requires localPath
   - ExtensionResponseSchema matching ExtensionRecord type

5. **supervisor/src/schemas/secret.ts** -- Secret Zod schemas:
   - CreateSecretSchema { name: string, value: string, category?: string, tags?: string[] }
   - UpdateSecretSchema { value?: string, category?: string, tags?: string[] }
   - SecretResponseSchema (NEVER includes the encrypted value -- only name, category, tags, timestamps)
   - SecretValueResponseSchema { name: string, value: string } (for getSecret only)

6. **supervisor/src/server.ts** -- Fastify server factory:
   - Export `buildServer(options: ServerOptions)` async function
   - Create Fastify instance with Pino logger, Zod type provider via `fastify-type-provider-zod`
   - Register @fastify/websocket
   - Register healthRoutes plugin at /api/v1
   - Expose `bootState` as a shared mutable reference that routes can read (set to "initializing" on start)
   - Return the server instance (do NOT call listen -- caller does that)

7. **supervisor/src/routes/health.ts** -- Health check route plugin:
   - Fastify plugin (using `fastify-plugin` wrapping pattern or Fastify's encapsulation)
   - GET /api/v1/health returns HealthResponse with current bootState, version from package.json, process uptime
   - Use Zod schema for response serialization

8. **supervisor/src/index.ts** -- Entry point (minimal for now):
   - Parse --dry-run from process.argv
   - Read CLAUDEOS_DATA_DIR env or default to /data
   - Call buildServer, then server.listen on port 3100
   - Log "Supervisor API listening on :3100"
   - If dry-run, log and return (don't proceed to boot sequence)
   - Graceful shutdown on SIGTERM/SIGINT

9. **supervisor/test/helpers/test-server.ts** -- Test helper:
   - Export `createTestServer()` that calls buildServer with a temp dataDir and isDryRun: true
   - Returns the Fastify instance for injection tests
   - Handles server close in afterAll
