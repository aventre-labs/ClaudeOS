<!-- VS Code Pseudoterminal API pattern from research -->

Pseudoterminal interface:
```typescript
interface Pseudoterminal {
  onDidWrite: Event<string>;       // fire to write to terminal
  onDidClose: Event<number | void>; // fire to close terminal
  onDidChangeName?: Event<string>;  // fire to update tab title
  open(initialDimensions: TerminalDimensions | undefined): void;
  close(): void;
  handleInput?(data: string): void; // receive user keystrokes
}
```

ExtensionTerminalOptions:
```typescript
vscode.window.createTerminal({
  name: string,
  pty: Pseudoterminal,
  iconPath: ThemeIcon,
  color: ThemeColor,
} as ExtensionTerminalOptions);
```

Key research insight -- input buffering:
- handleInput receives raw keystrokes character by character
- Buffer input until \r (Enter) is received, then flush to POST /sessions/:id/input
- Echo characters back to onDidWrite for local feedback
- This matches how Claude Code expects input: complete lines

    - SessionPseudoterminal implements vscode.Pseudoterminal
    - open() subscribes to session output via WsClient, loads initial scrollback from supervisor API, writes to terminal
    - open() does NOT write before it is called (pitfall 2 from research)
    - handleInput buffers characters, echoes to onDidWrite, flushes on \r to supervisor sendInput
    - handleInput handles backspace (\x7f): removes last char from buffer, writes \b \b to erase on screen
    - close() unsubscribes from WsClient
    - onDidChangeName fires when session name or status changes (name updated via terminal manager)
    - updateName(name) fires nameEmitter
    - onSessionExit() fires notification toast, keeps terminal open (does NOT fire closeEmitter)
    - TerminalManager.openTerminal(session) creates Pseudoterminal + vscode.Terminal, stores in Map
    - TerminalManager.openTerminal focuses existing terminal if session already open (no duplicate per user decision)
    - TerminalManager.getTerminal(sessionId) returns existing Terminal or undefined
    - TerminalManager.closeTerminal(sessionId) disposes terminal and removes from Map
    - TerminalManager handles terminal close event (user closes tab): cleans up from Map
    - TerminalManager.updateTerminalName(sessionId, name) calls Pseudoterminal.updateName
  </behavior>
  <action>
    1. Create src/terminal/session-terminal.ts -- SessionPseudoterminal class:
       - Constructor takes sessionId, supervisorClient, wsClient
       - writeEmitter = new vscode.EventEmitter<string>()
       - closeEmitter = new vscode.EventEmitter<number | void>()
       - nameEmitter = new vscode.EventEmitter<string>()
       - onDidWrite = writeEmitter.event
       - onDidClose = closeEmitter.event
       - onDidChangeName = nameEmitter.event
       - inputBuffer: string = ''
       - isOpen: boolean = false

       - open(initialDimensions?):
         - Set isOpen = true
         - Subscribe to output via wsClient.onOutput(sessionId, (data) => writeEmitter.fire(data))
         - wsClient.subscribe(sessionId)
         - Load initial scrollback: const output = await supervisorClient.getOutput(sessionId, true); if (output) writeEmitter.fire(output)
         - Wrap in try/catch -- if session not found, write error message to terminal

       - handleInput(data):
         - If data === '\r' (Enter): flush buffer to supervisorClient.sendInput(sessionId, inputBuffer), writeEmitter.fire('\r\n'), reset buffer
         - If data === '\x7f' (Backspace): if buffer.length > 0, remove last char, writeEmitter.fire('\b \b')
         - If data === '\x03' (Ctrl+C): send Ctrl+C to supervisor as special input, clear buffer
         - Else: append to buffer, writeEmitter.fire(data) for echo

       - close(): wsClient.unsubscribe(sessionId), dispose emitters

       - updateName(name: string): nameEmitter.fire(name)

       - onSessionExit(): writeEmitter.fire('\r\n[Session ended]\r\n'), show info notification via vscode.window.showInformationMessage

    2. Create src/terminal/terminal-manager.ts -- TerminalManager class:
       - Constructor takes supervisorClient, wsClient, sessionStore
       - terminals: Map<string, { terminal: vscode.Terminal, pty: SessionPseudoterminal }>
       - openTerminal(session: Session):
         - If terminals.has(session.id): focus existing terminal with terminal.show(), call sessionStore.markRead(session.id), return
         - Create new SessionPseudoterminal(session.id, supervisorClient, wsClient)
         - Create terminal via vscode.window.createTerminal({ name: session.name, pty, iconPath: STATUS_ICONS[session.status] })
         - Store in map
         - terminal.show()
         - sessionStore.markRead(session.id)
         - Listen for vscode.window.onDidCloseTerminal to clean up map when user closes tab
       - focusTerminal(sessionId): find and show
       - closeTerminal(sessionId): dispose terminal, remove from map
       - updateTerminalName(sessionId, name): call pty.updateName(name)
       - notifySessionExit(sessionId): call pty.onSessionExit(), show toast "Session {name} has ended"
       - dispose(): close all terminals
