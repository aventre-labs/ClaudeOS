1. **supervisor/src/services/secret-store.ts** -- SecretStore class:
   - Constructor takes dataDir. Reads encryption key from /data/config/auth.json. Loads existing secrets from /data/secrets/secrets.json.
   - `encrypt(plaintext)`: Generate random 12-byte IV, create AES-256-GCM cipher with master key + IV, encrypt, return { encrypted (hex), iv (hex), authTag (hex) }. Per research code example.
   - `decrypt(encrypted, ivHex, authTagHex)`: Create decipher, set auth tag, decrypt, return plaintext.
   - `set(name, value, category?, tags?)`: Encrypt value, store as SecretEntry in secrets.json. Atomic write (temp file + rename).
   - `get(name)`: Find entry, decrypt, return plaintext value. Throw if not found.
   - `has(name)`: Check if name exists in store.
   - `delete(name)`: Remove from store, atomic write.
   - `list()`: Return all secrets as Secret[] (name, category, tags, timestamps) -- NEVER include encrypted value.
   - `generateMasterKey()`: Static method. Generate 32 random bytes (256-bit). Return as hex string. Used during first boot.
   - Thread safety: Read/write serialization via a simple mutex (or sequential promise chain) to prevent corruption on concurrent writes.

2. **supervisor/src/services/extension-installer.ts** -- ExtensionInstaller class:
   - Constructor takes dataDir, logger. Loads install state from /data/extensions/install-state.json.
   - `installFromGitHub(repo, tag)`:
     1. Set state to "downloading"
     2. Use `fetch()` (Node 22 has global fetch) to call GitHub API: `https://api.github.com/repos/{repo}/releases/tags/{tag}`
     3. Find VSIX asset in release assets (file ending in .vsix)
     4. Download VSIX to temp directory
     5. Set state to "installing"
     6. Run `code-server --install-extension {vsix_path}` via execFile
     7. Set state to "installed" with installedAt timestamp
     8. On failure at any step: set state to "failed" with error message
   - `installFromSource(localPath)`:
     1. Set state to "installing"
     2. Run `npm install && npm run compile && npm run package` in localPath via execFile
     3. Find .vsix file in localPath
     4. Run `code-server --install-extension {vsix_path}`
     5. Set state to "installed" or "failed"
   - `installFromVsix(vsixPath)`:
     1. Validate file exists and ends in .vsix
     2. Set state to "installing"
     3. Run `code-server --install-extension {vsixPath}`
     4. Set state to "installed" or "failed"
   - `getInstallState()`: Return all ExtensionRecord entries
   - `getPendingExtensions()`: Return extensions in pending/downloading/failed states (for retry on boot)
   - Per-extension install tracking persisted to /data/extensions/install-state.json (locked decision)

3. **supervisor/src/services/settings-store.ts** -- SettingsStore class:
   - Constructor takes dataDir. Loads from /data/config/settings.json or creates defaults.
   - Default settings: { reloadBehavior: "force" } (locked decision: defaults to force reload after extension install)
   - `get()`: Return current settings object
   - `update(partial)`: Deep merge partial into settings, persist to disk
   - Simple JSON file persistence with atomic writes

4. **Tests**:

   - extension-installer.test.ts: Test state transitions (pending->downloading->installing->installed). Mock fetch and execFile. Test GitHub API URL construction. Test failure sets state to "failed" with error. Test getPendingExtensions returns failed extensions for retry.

1. **supervisor/src/services/boot.ts** -- BootService class:
   - Constructor takes dataDir, logger, and references to ExtensionInstaller and server's bootState setter
   - `isConfigured()`: Check if /data/config/auth.json exists
   - `serveSetupPage(port)`:
     1. Create a minimal HTTP server (or use Fastify instance) serving first-boot/setup.html on the given port
     2. Register POST /api/v1/setup that:
        a. Receives { password, confirmPassword }
        b. Validates passwords match and meet minimum length (8 chars)
        c. Hashes password with Node.js crypto.scryptSync (per research: use proper KDF, not plain hash)
        d. Generates random 256-bit master encryption key via crypto.randomBytes(32)
        e. Stores { passwordHash (hex), salt (hex), encryptionKey (hex) } in /data/config/auth.json
        f. Sets bootState to "installing"
        g. Returns { success: true }
     3. Block (return a Promise) until POST /api/v1/setup completes
     4. After password stored, close the setup page HTTP handler
   - `installExtensions()`:
     1. Read config/default-extensions.json (empty array in Phase 1)
     2. For each extension: call extensionInstaller.installFromGitHub(repo, tag)
     3. Fail-fast: if any install fails, halt and throw (locked decision)
     4. On next boot: getPendingExtensions() finds failed ones, retries only those
     5. Log progress to stdout (user sees in terminal during install -- locked decision)
     6. Set bootState to "ready" when complete
   - `getStoredPassword()`: Read /data/config/auth.json, return the raw password (stored hash is for verification, but code-server needs the plaintext password -- actually, re-read the flow: the first-boot page receives the plaintext password. We need to store the plaintext for code-server's --auth. Store it encrypted with the master key.)
     - IMPORTANT: code-server needs the plaintext password for `--auth password`. Store the password encrypted with the master key in auth.json, decrypt it at boot time to pass to code-server.
   - `startCodeServer(options)`:
     1. Spawn code-server as child process with:
        - `--bind-addr 0.0.0.0:{port}` (default 8080)
        - `--auth password` + environment PASSWORD={password}
        - Product.json path for branding (SUP-01)
        - User data dir on persistent volume
     2. Set bootState to "ok"
     3. Handle code-server crash (restart or log error)
