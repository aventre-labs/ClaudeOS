    1. **Function signature:** `export function registerInstallCommand(context: vscode.ExtensionContext, client: SupervisorClient, outputChannel: vscode.OutputChannel): vscode.Disposable`

    2. **registerCommand handler flow:**

       **Step 1 -- Pick install method:**
       ```typescript
       const method = await vscode.window.showQuickPick([
         { label: "From GitHub Release", value: "github-release", description: "Install from a GitHub release VSIX asset" },
         { label: "From Local Source", value: "build-from-source", description: "Build and install from a local directory" },
         { label: "From VSIX File", value: "local-vsix", description: "Install a pre-built .vsix file" },
       ], { placeHolder: "Select install method" });
       if (!method) return;
       ```

       **Step 2 -- Collect method-specific inputs:**

       For `github-release`:
       - Prompt for repo via showInputBox: "GitHub repository (owner/repo)", placeholder "aventre-labs/claudeos-memory"
       - Prompt for tag via showInputBox: "Release tag", placeholder "v0.1.0"
       - Auto-detect PAT: try `vscode.extensions.getExtension<SecretsPublicApi>("claudeos.claudeos-secrets")`, if active call `api.listSecrets()`, filter for secrets where `category === "github-pat"` or `name.toLowerCase().includes("github")`. If exactly 1 PAT found, use it automatically. If >1, show QuickPick to let user choose. If 0 or secrets extension unavailable, skip PAT (public repo assumed).
       - If PAT selected, pass it to the supervisor as `secretName` parameter in the install body. NOTE: The supervisor's current installFromGitHub does not yet accept a secretName param. For now, include `secretName` in the request body -- the supervisor will ignore it if not implemented, and this is forward-compatible for when it is implemented.

       For `build-from-source`:
       - Show folder picker via `vscode.window.showOpenDialog({ canSelectFolders: true, canSelectFiles: false, canSelectMany: false, openLabel: "Select Extension Directory" })`
       - Use `selected[0].fsPath` as localPath

       For `local-vsix`:
       - Show file picker via `vscode.window.showOpenDialog({ canSelectFiles: true, canSelectFolders: false, canSelectMany: false, filters: { "VSIX": ["vsix"] }, openLabel: "Select VSIX File" })`
       - Use `selected[0].fsPath` as localPath

       If any input returns undefined (user cancelled), return early.

       **Step 3 -- Execute install with progress:**
       ```typescript
       await vscode.window.withProgress(
         { location: vscode.ProgressLocation.Notification, title: `Installing extension...`, cancellable: false },
         async (progress) => {
           progress.report({ message: "Sending install request..." });
           outputChannel.appendLine(`[${new Date().toISOString()}] Installing via ${method.value}: ${repo ?? localPath}`);
           outputChannel.show(true); // Reveal without focus

           try {
             const result = await client.installExtension({ method: method.value, repo, tag, localPath });
             if (result.state === "installed") {
               outputChannel.appendLine(`[${new Date().toISOString()}] Installed: ${result.name} v${result.version}`);
               vscode.window.showInformationMessage(`Extension "${result.name}" installed successfully.`);
             } else if (result.state === "failed") {
               outputChannel.appendLine(`[${new Date().toISOString()}] FAILED: ${result.error}`);
               vscode.window.showErrorMessage(`Extension install failed: ${result.error}`);
             }
           } catch (err) {
             const msg = err instanceof Error ? err.message : String(err);
             outputChannel.appendLine(`[${new Date().toISOString()}] ERROR: ${msg}`);
             vscode.window.showErrorMessage(`Extension install error: ${msg}`);
           }
         }
       );
       ```
