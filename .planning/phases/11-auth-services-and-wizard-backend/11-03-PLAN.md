      anthropicAuth: AnthropicAuthService;
      secretStore: SecretStore | null;

      options: WizardRouteOptions,
    ): Promise<void> { ... }

    Implementation details:

    **Rate limiting:** Register `@fastify/rate-limit` scoped to wizard routes: `{ max: 30, timeWindow: 60000 }`.

    **SSE client management:**
    - `sseClients: Map<string, ServerResponse>` tracks connected clients
    - `broadcastEvent(event: string, data: unknown)` writes to all clients in SSE format: `event: {event}\ndata: {JSON}\n\n`
    - Heartbeat interval (15s) per client, cleaned up on close
    - `request.raw.on("close", ...)` removes client and clears heartbeat

    **Completion guard (preHandler hook):**
    - Check `wizardState.isCompleted()` for all routes except GET /wizard/status
    - Return 410 with `{ error: "Wizard already completed", statusCode: 410 }`

    **GET /wizard/status:**
    - Returns `wizardState.getState()`

    **POST /wizard/railway/start:**
    - Check `railwayAuth.isRunning()` — if true, return 409
    - Call `railwayAuth.startLogin()` with callbacks:
      - onPairingInfo: `broadcastEvent("railway:started", { pairingCode, url })`
      - onComplete: if success, optionally extract + store token, mark step complete, `broadcastEvent("railway:complete", result)` + `broadcastEvent("wizard:step-completed", ...)`
    - Return 202 `{ message: "Railway login started" }`

    **POST /wizard/anthropic/key:**
    - Validate body with AnthropicKeyBodySchema
    - Call `anthropicAuth.validateApiKey(apiKey)`
    - If invalid: return 400 with error
    - If valid and secretStore available: call `anthropicAuth.storeApiKey(apiKey, secretStore)`
    - Mark anthropic step complete with method "api-key"
    - broadcastEvent("anthropic:key-validated", { success: true })
    - broadcastEvent("wizard:step-completed", { step: "anthropic", ... })
    - Return 200

    **POST /wizard/anthropic/login:**
    - Check `anthropicAuth.isRunning()` — if true, return 409
    - Call `anthropicAuth.startClaudeLogin()` with callbacks:
      - onLoginUrl: `broadcastEvent("anthropic:login-started", { url })`
      - onComplete: if success, mark step complete with method "claude-login", broadcast events
    - Return 202

    **GET /wizard/events:**
    - Set response headers: Content-Type: text/event-stream, Cache-Control: no-cache, Connection: keep-alive
    - Use `reply.raw.writeHead(200, headers)` then `reply.raw.write()` for SSE
    - Send `event: connected\ndata: {...}\n\n` immediately
    - Register client in sseClients Map
    - Start 15s heartbeat: `reply.raw.write(": heartbeat\n\n")`
    - On close: cleanup
    - Return `reply` without calling `reply.send()`

    **POST /wizard/complete:**
    - Call `wizardState.complete()`
    - If error (steps incomplete): return 400
    - broadcastEvent("wizard:completed", { completedAt })
    - Return 200 with WizardCompleteResponseSchema
