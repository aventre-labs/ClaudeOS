<objective>
Implement Railway and Anthropic auth services that manage CLI subprocess flows and API key validation, with SecretStore integration for credential persistence.

Purpose: These services encapsulate the auth logic that wizard routes (Plan 03) will call. Railway auth spawns `railway login --browserless` as a child process. Anthropic auth validates API keys via HTTP and spawns `claude login` as an alternative flow. Both emit events for SSE notification.
Output: auth-railway.ts, auth-anthropic.ts with full test coverage.
</objective>
