# Phase 14 -- UI Design Contract

> Visual and interaction contract for the Theme Foundation & Infrastructure phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (VS Code webview inline CSS in TypeScript template literals) |
| Icon library | VS Code Codicons (built into VS Code webviews) |
| Font | `var(--vscode-font-family)` for body, `var(--vscode-editor-font-family)` for code (no custom fonts in VS Code panels) |

**Note:** This phase targets VS Code extension webviews, not a standalone web application. All styling is inline CSS within TypeScript template literals. There is no external CSS build pipeline, no Tailwind, and no component library. The wizard is a separate React+Vite app with its own fonts (Syne, Figtree, JetBrains Mono) and is explicitly out of scope for theme variable migration per CONTEXT.md.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status badge margins, inline padding |
| sm | 8px | Compact element spacing, grid gaps between shortcut cards, inner card margins |
| md | 16px | Default element spacing, card padding, form group margins, section heading bottom margin |
| lg | 24px | Section padding (vertical), form detail area padding, hero tagline bottom margin, action bar top margin |
| xl | 32px | Section padding (horizontal), hero horizontal padding |
| 2xl | 48px | Hero vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions: none

**Source:** Derived from existing Home panel CSS (`padding: 48px 32px` for hero, `24px 32px` for sections, `16px` card padding, `12px` grid gaps, `8px` margin-bottom) -- the existing codebase already follows an 8-point grid with 4px subdivisions. This contract codifies the existing pattern.

## Typography

All sizes are delivered via `var(--vscode-font-size)` for body text. The following roles define overrides applied as explicit `font-size` values in the inline CSS.

| Role | Size | Weight | Line Height | VS Code Variable / Literal |
|------|------|--------|-------------|---------------------------|
| Body | 13px | 400 (regular) | 1.5 | `var(--vscode-font-size)` (VS Code default is 13px) |
| Label / Meta | 12px | 400 (regular) | 1.4 | Literal `12px` (session card meta, shortcut labels, timestamps -- differentiated from Body by smaller size alone) |
| Section Heading | 16px | 600 (semibold) | 1.2 | Literal `16px` (section h2 elements) |
| Display / Hero | 28px | 600 (semibold) | 1.1 | Literal `28px` (welcome page hero heading -- differentiated from Section Heading by larger size alone) |

**Weights used:** 400 (regular) and 600 (semibold). Body and Label/Meta share weight 400, differentiated by size (13px vs 12px). Section Heading and Display/Hero share weight 600, differentiated by size (16px vs 28px).

**Source:** Existing Home panel uses `font-size: 14px` for labels and `16px` for h2 (RESEARCH.md). VS Code's default font size is 13px. Wizard uses `line-height: 1.6` for body -- VS Code panels use 1.5 to match the tighter VS Code UI density.

## Color

### Theme Palette (via `workbench.colorCustomizations` in settings.json)

The full ClaudeOS Dark theme is defined as a `workbench.colorCustomizations` overlay on top of "Default Dark Modern". The gold accent `#d4a054` replaces the current purple `#c084fc` to unify with the wizard brand.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0e0e10` | Editor background, main canvas surface |
| Secondary (30%) | `#0c0c0e` sidebar, `#131315` cards/tabs | Sidebar background, card surfaces, active tab background, notifications |
| Accent (10%) | `#d4a054` | See "Accent reserved for" list below |
| Destructive | `#e05252` | Delete secret button, error-state borders |

### Accent Reserved For

The accent color `#d4a054` (gold) is applied ONLY to:

1. **Activity bar foreground** -- active icon color (`activityBar.foreground`)
2. **Activity bar badge** -- notification count background (`activityBarBadge.background`)
3. **Tab active border top** -- the top-border highlight on the active editor tab (`tab.activeBorderTop`)
4. **Button primary background** -- primary action buttons (`button.background`)
5. **Focus border** -- keyboard focus ring on all interactive elements (`focusBorder`)
6. **Editor cursor** -- blinking cursor color (`editorCursor.foreground`)
7. **Card hover border** -- session/shortcut card hover state (webview CSS: `border-color` on `:hover`)
8. **Active selection left border** -- secret list active item indicator (webview CSS: `border-left-color`)
9. **Input focus border** -- form field focus ring (webview CSS: `border-color` on `:focus`)
10. **Radial ambient glow** -- decorative background effect using `rgba(212, 160, 84, 0.04)` (hardcoded rgba, not variable -- glow is decorative)

The accent is NEVER used for: body text, card backgrounds, status badges, meta text, section headings, or general borders.

### Semantic Status Colors (webview only, hardcoded)

| Status | Color | Usage |
|--------|-------|-------|
| Active | `#22c55e` | Session active status badge |
| Idle | `#facc15` | Session idle status badge |

| Stopped | `#6b7280` | Session stopped status badge |
| Zombie | `#ef4444` | Session zombie status badge |

**Note on status badge colors:** These are hardcoded hex values, not theme variables. Status badges are semantic indicators with fixed meaning -- they do NOT change with the VS Code theme. This is intentional.

## Webview Visual Effects

### Noise Texture Overlay

Applied to Home panel webview body via `::before` pseudo-element.

```css
body::before {
  content: "";
  position: fixed;
  inset: 0;
  background-image: url("data:image/svg+xml,%3Csvg viewBox='0 0 256 256' xmlns='http://www.w3.org/2000/svg'%3E%3Cfilter id='n'%3E%3CfeTurbulence type='fractalNoise' baseFrequency='0.9' numOctaves='4' stitchTiles='stitch'/%3E%3C/filter%3E%3Crect width='100%25' height='100%25' filter='url(%23n)' opacity='0.025'/%3E%3C/svg%3E");
  pointer-events: none;
  z-index: 0;
}
```

**CSP requirement:** Add `data:` to `img-src` directive: `img-src ${cspSource} data:;`

### Radial Ambient Glow

Applied to Home panel webview body via `::after` pseudo-element.

```css
body::after {
  content: "";
  position: fixed;
  top: 30%;
  left: 50%;
  width: 800px;
  height: 600px;
  transform: translate(-50%, -50%);
  background: radial-gradient(
    ellipse at center,
    rgba(212, 160, 84, 0.04) 0%,
    transparent 70%
  );
  pointer-events: none;
  z-index: 0;
}
```

**Note:** The glow uses hardcoded `rgba(212, 160, 84, 0.04)` -- NOT a `var(--vscode-*)` variable. VS Code theme variables resolve to opaque hex and cannot be used with opacity in gradients. This is a decorative effect and is acceptable as a hardcoded value per RESEARCH.md recommendation.

### Focal Point

Primary focal point: Hero heading (`ClaudeOS` at 28px) anchored by the radial ambient glow.

### Where Effects Apply

| Panel | Noise Texture | Radial Glow | Rationale |
|-------|--------------|-------------|-----------|
| Home (welcome page) | Yes | Yes | Branded premium feel, matches wizard aesthetic |
| Secrets | No | No | Utility panel -- clean, distraction-free |

## CSS Migration Rules

These rules govern the conversion of existing webview CSS from hardcoded values to pure `var(--vscode-*)` CSS variables.

### BEFORE (current pattern -- must be eliminated)

```css
.session-card {
  background: var(--vscode-sideBar-background, var(--vscode-editor-background));
  border: 1px solid var(--vscode-panel-border, #333);
}
```

### AFTER (target pattern -- required for all VS Code webview CSS)

```css
.session-card {
  background: var(--vscode-sideBar-background);
  border: 1px solid var(--vscode-panel-border);
}
```

### Migration Checklist

1. Remove ALL `--claudeos-accent`, `--claudeos-accent-hover`, `--claudeos-gradient-start`, `--claudeos-gradient-end` custom CSS variables
2. Replace `var(--claudeos-accent)` with `var(--vscode-focusBorder)` for focus/hover states, or `var(--vscode-button-background)` for primary buttons
3. Strip ALL hardcoded hex fallback values from `var()` calls (e.g., `var(--vscode-panel-border, #333)` becomes `var(--vscode-panel-border)`)
4. Replace hardcoded `rgba(255,255,255,0.2)` button backgrounds with `var(--vscode-button-background)` / `var(--vscode-button-foreground)`
5. Replace hero gradient (`linear-gradient(135deg, ...)`) with solid `var(--vscode-sideBar-background)` or remove the gradient entirely in favor of theme-variable-only styling
6. Ensure `font-family` uses `var(--vscode-font-family)` only -- no system-ui fallback stacks

### Variable Mapping Reference

| Old Pattern | New Pattern |
|-------------|-------------|
| `--claudeos-accent` / `#c084fc` | `var(--vscode-focusBorder)` or `var(--vscode-button-background)` |
| `--claudeos-accent-hover` / `#a855f7` | `var(--vscode-button-hoverBackground)` |
| `var(--vscode-panel-border, #333)` | `var(--vscode-panel-border)` |
