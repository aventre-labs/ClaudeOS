### Remaining Non-Critical Integration Gaps

#### INT-04: default-extensions.json is empty

- **From:** Phase 1 (BootService.installExtensions) **To:** ClaudeOS extension artifacts
- **Affects:** SUP-07, SUP-08, DEP-02 (operationally, not functionally)
- **Issue:** `config/default-extensions.json` contains `[]`. The install pipeline works correctly (SUP-07 satisfied), but no extensions are listed for auto-install. A freshly deployed container will have the supervisor running but no VS Code UI extensions loaded.
- **Note:** This is an operational/deployment concern, not a requirement gap. The install pipeline capability is verified. Extensions can be pre-baked into the container image during the Nix build or added to the file before deployment.

#### INT-05: PAT detection silently degrades

- **From:** Phase 4 (install-extension.ts) **To:** Phase 3 (claudeos-secrets)
- **Affects:** IMP-03 (edge case only)
- **Issue:** `detectGitHubPat()` checks `secretsExt.isActive` and returns `undefined` if the secrets extension hasn't been activated yet. PAT-based private repo installs silently skip the PAT.
- **Note:** Graceful degradation by design. The user can still manually configure auth. Alternatively, calling `secretsExt.activate()` would fix this.
